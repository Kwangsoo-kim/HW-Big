// subquery
	

	1. 서브쿼리란 - 하나의 sql 문장의 절 안에 포함된 또 하나의 select 문자
		      - 반드시 괄호안에 넣어야한다.
		      - 메인 쿼리 실행전에 한번만 실행된다
	
	2. 단일행 서브쿼리 - 수행 결과가 오직 하나의 행만을 반환하는 서브쿼리
			- 서브쿼리에서 그룹함수 사용 가능하고 다중열 서브쿼리도 가능
			- 연산자 : =,>,>=,<,<=,!=
	2. 다중행 서브쿼리 - 서브쿼리에서 반환되는 결과가 하나 이상의 행일 때 
			- 반드시 다중 행 연산자와 함께 사용 
			- 연산자 - IN : 메인 쿼리의 비교 조건(‘=’ 연산자로 비교할 경우)이 
				서브 쿼리의 결과 중에서 하나라도 일치하면 참
				- ANY, SOME : 메인 쿼리의 비교 조건이 서브 쿼리의 
				검색 결과와 하나 이상이 일치하면 참	
				- ALL : 메인 쿼리의 비교 조건이 서브 쿼리의 검색 결과와 
				모든 값이 일치하면 참
				- EXISTS : 메인 쿼리의 비교 조건이 서브 쿼리의 
				결과 중에서 만족하는 값이 하나라도 존재하면 참



	** IN 사용시 앞에 NOT을 사용할땐 주의해야한다. NULL값이 있는 데이터가 있다면 인식하지 못한다. 
			따라서 NOT IN 사용시에는 NULL을 WHERE 절을 사용해서 빼고 사용해야한다. 
			NULL은 IS NULL, IS NOT NULL로만 가능
