// View_Top-N
	
	View	- 행과 컬럼으로 구성된 가상테이블이며, 제한적 접근을 위함.
		- 기본 테이블의 데이터가 변경되면 View에도 반영되고, View에서 변경
		하면 기본 테이블의 데이터도 변경됨.
		- 제한조건 - with read olny 설정한 View는 데이터 갱신 불가
			  - with  check option을 설정한 View는 View의 where절에 해당
			되는 데이터만 DML문 가능.
		- VIEW가 다음을 포함한다면 행을 제거하거나 수정하거나 추가할 수 없다.
			그룹함수
			GROUP BY 절
			DISTINCT 키워드

		1. 단순뷰	- 하나의 테이블로 구성됨
			- 기본 테이블의 데이터 조작 가능
			- 함수 그룹데이터 사용가능
		2. 복합뷰	- 하나 이상의 테이블로 구성
			- DML문 제한적 사용가능
			- 함수 그룹데이터 사용가능

	Inline View - From절 상의 서브쿼리

	Top-N	- Rownum 사용, 데이터를 정렬하는 질의를 서브쿼리나 inline view로 생성
		- from절 내에 서브쿼리가 필요하며 서브쿼리내에 order by절이 필요하다.
		- 일반적으로 서브쿼리 절 내에 서브쿼리절을 삽입하여 첫번째 서브쿼리 절에 
		rownum을 정의, 마지막 sql문에서 rank별로 추출가능하게 한다.
